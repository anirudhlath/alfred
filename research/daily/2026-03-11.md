## Key Observations

1. **Latency regression.** The 2.2x increase in median latency vs March 10 evening warrants investigation. Need to separate the contribution of (a) larger prompts, (b) hardware thermals, and (c) Ollama state.
2. **Prompt size correlation.** There is a visible (though not perfectly linear) relationship between prompt tokens and latency. The two highest-latency runs (10453 ms / 1106 tokens, 10132 ms / 600 tokens) suggest prompt size is not the only factor -- the 600-token run at 10132 ms points to infrastructure overhead.
3. **Completion stability.** All 8 runs produced exactly 27 completion tokens. The structured output constraint is working perfectly.
4. **Concurrent events.** Runs 10-11 (01:25) and runs 12-17 (02:59-03:00) have near-identical timestamps, suggesting multiple events fired concurrently. This may cause contention on Ollama inference (single model instance, sequential processing).

## Anomalies

- **Run 13 (600 prompt tokens, 10132 ms):** Unexpectedly high latency for a small prompt. May indicate queuing delay behind run 12 (1093 tokens), since both timestamps are within 1 second of each other.
- **Runs with paired timestamps** (10/11, 12/13, 14/15, 16/17): Events appear to arrive in pairs. If Ollama processes them sequentially, the second event in each pair would include queuing time in its E2E measurement.
